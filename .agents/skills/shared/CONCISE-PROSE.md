# Omit Needless Words

Shared writing principle. Reference this from any skill whose
deliverable is prose a human reads — docs, PR/commit messages,
changelog entries, plans, research, summaries, handoffs, ADR/PRD text.

Omit needless words (Strunk & White). Write the point, then cut: delete
any word, sentence, or qualifier that survives removal without loss of
meaning. Prefer the shorter word, active voice, one clause over two.

- Cut filler and throat-clearing ("this PR", "simply", "just", "in
  order to", "it should be noted that").
- Cut hedging and unnecessary qualifiers.
- Don't restate what a heading, table, or diagram already says.
- Keep technical accuracy exact — concision never drops a load-bearing
  detail (path, flag, constraint, edge case). When tightening risks
  losing one, keep it.
